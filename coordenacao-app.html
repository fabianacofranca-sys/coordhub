⚙️ Configurações
☁️ Sincronização Multi-Dispositivo — Firebase
⚪ Não configurado — dados salvos localmente
📋 Como configurar — passo a passo (gratuito, ~10 min)
1. Acesse console.firebase.google.com → Criar projeto → dê um nome → Continuar (pode desativar Analytics)
2. Após criar, clique no ícone </> (Web) → dê um apelido → Registrar app
3. Copie todo o bloco const firebaseConfig = { … } e cole no campo abaixo
4. Menu lateral: Authentication → Começar → Google → Ativar → Salvar
5. Menu lateral: Firestore Database → Criar banco de dados → Modo produção → Avançar → Ativar
6. Aba Regras do Firestore — substitua tudo por:
rules_version = '2';
service cloud.firestore {
  match /databases/{database}/documents {
    match /coordhub/{userId}/{document=**} {
      allow read, write: if request.auth != null
        && request.auth.uid == userId;
    }
  }
}
7. Clique em Publicar.
⚠ Para login com Google a partir do arquivo local, abra o Terminal na pasta do arquivo e execute:
python -m http.server 8000 Depois acesse http://localhost:8000/coordenacao-app.html
Python já vem no Windows 10/11. Ou hospede no GitHub Pages para acesso permanente de qualquer dispositivo.
💾 Backup e Restauração de Dados

Seus dados ficam no armazenamento local do navegador (localStorage). Exporte um backup antes de qualquer atualização do arquivo ou limpeza do navegador.

⬇ Exportar Backup Completo
Gera um arquivo .json com todos os seus dados: atividades, FCA, scorecard, saúde do curso, BIs, configurações, etc.
⬆ Restaurar Backup
Importa um backup exportado anteriormente. Atenção: substitui todos os dados atuais.
Rotina recomendada: exporte um backup antes de pedir qualquer melhoria ou atualização do arquivo. O arquivo HTML pode ser atualizado livremente — os dados não são perdidos, desde que o arquivo permaneça no mesmo caminho e o cache do navegador não seja limpo.
